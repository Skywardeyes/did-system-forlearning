# DID/VC 本地演示系统测试与人工验收

> 生命周期扩展后，自动化覆盖 `did:example`/`did:key`、DID 更新/轮换/停用、VC 暂停/恢复/替代/撤销、七项验签、列表查询及六条 E2E 旅程。

## 1. 自动化测试

执行命令：

```bash
npm test
```

测试环境：Node.js 20 或更高版本。每条测试使用独立临时目录，不读取或修改演示数据。

日志专项自动化文件：`log-store.test.js`、`log-service.test.js`、`audit-logging.test.js`、`log-api.test.js` 和 `log-ui.test.js`，覆盖脱敏、留存、审计 action、关联 ID、系统事件、日志 API 与级别配色。

| 编号 | 测试场景 | 预期结果 | 当前结果 |
|---|---|---|---|
| T01 | 创建 DID 并读取公开状态 | DID Document 完整，接口不含私钥 | 通过 |
| T02 | 验证原始 VC | 七项检查全部通过 | 通过 |
| T03 | 修改姓名和课程 | 签名检查失败，总体失败 | 通过 |
| T04 | 修改为未知 Issuer DID | Issuer 解析失败，总体失败 | 通过 |
| T05 | 签发已过期 VC | 签名通过、有效期失败、总体失败 | 通过 |
| T06 | 撤销原始 VC | 签名通过、撤销检查失败、总体失败 | 通过 |

本轮自动化执行结果以 `npm test` 的实时输出为准，要求 0 failed、0 skipped、0 todo。

## 2. 人工验收前准备

1. 在项目根目录执行 `npm start`。
2. 浏览器打开 <http://127.0.0.1:4173>。
3. 点击左下角“重置并载入演示数据”。
4. 确认页面右上角显示本地时间，操作台可正常切换。

## 3. 人工验收脚本

| 步骤 | 操作 | 预期结果 | 验收记录 |
|---|---|---|---|
| A01 | 打开“DID 身份” | 可见一个 Issuer 和一个 Holder | 待签字 |
| A02 | 查看 Issuer DID Document | 包含 id、verificationMethod、authentication、assertionMethod | 待签字 |
| A03 | 打开“凭证签发”并提交表单 | 生成包含 issuer、subject 和 proof 的 VC JSON | 待签字 |
| A04 | 打开“凭证验证”，载入最新凭证并验证 | 七项检查全部通过 | 待签字 |
| A05 | 点击“模拟篡改姓名”并验证 | 签名检查失败，总体失败 | 待签字 |

| A08 | 再次验证原始凭证 | 签名通过、撤销状态失败、总体失败 | 待签字 |
| A09 | 刷新浏览器 | DID、凭证和验证记录仍然存在 | 待签字 |
| A10 | 打开日志中心 | 可见 audit/system、成功/失败、INFO/WARN/ERROR 记录 | 待签字 |

| A12 | 查看日志详情 | 有 correlationId，且无私钥、proofValue、完整 VC 或 token | 待签字 |
| A13 | 重置演示数据 | 旧日志保留并新增 DEMO_RESET | 待签字 |
| A14 | 确认清空日志 | 删除旧日志，仅保留 LOG_CLEAR 摘要 | 待签字 |
