# DID/VC 本地演示系统测试与人工验收

> 生命周期扩展后，自动化覆盖 `did:example`/`did:key`、DID 更新/轮换/停用、VC 暂停/恢复/替代/撤销、七项验签、列表查询及六条 E2E 旅程。

## 1. 自动化测试

执行命令：

```bash
npm test
```

测试环境：Node.js 20 或更高版本。每条测试使用独立临时目录，不读取或修改演示数据。

日志专项自动化文件：`log-store.test.js`、`log-service.test.js`、`audit-logging.test.js`、`log-api.test.js` 和 `log-ui.test.js`，覆盖脱敏、留存、审计 action、关联 ID、系统事件、日志 API 与级别配色。

| 编号 | 测试场景 | 预期结果 | 当前结果 |

| T01 | 创建 DID 并读取公开状态 | DID Document 完整，接口不含私钥 | 通过 |

## 2. 人工验收前准备

1. 在项目根目录执行 `npm start`。
2. 浏览器打开 <http://127.0.0.1:4173>。
3. 点击左下角“重置并载入演示数据”。
4. 确认页面右上角显示本地时间，操作台可正常切换。

## 6. Agent 可执行的分层测试

| 层级 | 命令 | 主要范围 |
|---|---|---|
| 单元 | `npm run test:unit` | 加密、DID Method、查询、脱敏、UI 纯函数 |
| 集成 | `npm run test:integration` | 临时 JSON 存储、日志、DID/VC 服务协作 |
| 接口 | `npm run test:api` | 全部 HTTP 路由、状态码、错误和分页契约 |
| 功能 | `npm run test:functional` | 四种 Method 组合及完整生命周期旅程 |
| 安全 | `npm run test:security` | 泄密、XSS、原型污染、路径穿越和协议加固 |
| UI | `npm run test:ui` | Chromium 导航、表单、生命周期、日志和窄屏布局 |

首次执行 UI 测试前运行：

```bash
npm install
npx playwright install chromium
```

全量命令 `npm run test:all` 会依次运行全部 Node 测试和 Chromium UI 测试，即使 Node 层失败也不会跳过 UI，最后根据所有结果统一设置退出码。

执行环境基线：

- Node.js 20.18.0
- npm 10.8.2
- `@playwright/test` 1.61.1
- Chromium 149.0.7827.55（Playwright revision 1228）

测试运行数据全部位于系统临时目录；`test-results/` 保存失败截图、视频和 trace，`playwright-report/` 保存 HTML 报告。

每次全量运行还会创建本地 `test-records/YYYY-MM-DD/<带时区的测试时间>/` 目录：

- `result.md`：测试时间、阶段统计、通过率、失败清单和简要分析。
- `metadata.json`：Git、环境、时间、耗时、退出码和结构化统计。
- `node.log`：Node 测试完整输出。
- `ui.log`：Chromium UI 测试完整输出。

`test-records/` 不提交 Git。修复后基线为 Node 114/114、Chromium UI 6/6，总计 120/120 通过。缺陷复现、修复提交与回归证据见 `docs/测试缺陷报告.md`。
