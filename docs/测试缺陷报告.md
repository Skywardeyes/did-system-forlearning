# 测试缺陷报告

执行基线：Node.js 20.18.0，npm 10.8.2，`@playwright/test` 1.61.1。以下问题均在隔离临时数据环境中连续复现，并已根据失败测试完成修复和全量回归。

## DEF-001：Base58 前导零编码多输出一个字符

- 风险级别：中
- 状态：已修复
- 修复提交：`a85b536`
- 修复时间：2026-07-10 21:55（Asia/Shanghai）
- 测试命令：`npm run test:unit`
- 失败用例：`base58 encoding follows empty, leading-zero and byte vectors`
- 期望结果：单字节 `0x00` 编码为 `1`，双字节 `0x0000` 编码为 `11`。
- 实际结果：单字节 `0x00` 编码为 `11`。
- 影响：通用 Base58 编码结果不符合标准向量；当前 did:key 的 multicodec 前缀不以零字节开头，因此主流程暂未触发。
- 建议：当数值部分为零时不要额外追加兜底字符，只保留按前导零数量生成的 `1`。
- 回归结果：目标用例及 `npm run test:unit` 全部通过。

## DEF-002：并发 JSON 状态更新会丢失写入

- 风险级别：高
- 状态：已修复
- 修复提交：`a85b536`
- 修复时间：2026-07-10 21:55（Asia/Shanghai）
- 测试命令：`npm run test:integration`
- 失败用例：`concurrent service writes preserve both identities`
- 期望结果：两个并发 DID 创建完成后存在两条身份记录。
- 实际结果：最终仅保留一条记录。
- 影响：并发请求可能互相覆盖，造成 DID、VC 或验证记录丢失。
- 建议：将读取、变更和写入整个事务串入同一队列，不能只串行化最终 `save`。
- 回归结果：目标用例及 `npm run test:integration` 全部通过。

## DEF-003：JSON 写接口不校验 Content-Type

- 风险级别：中
- 状态：已修复
- 修复提交：`1d29be4`
- 修复时间：2026-07-10 21:55（Asia/Shanghai）
- 测试命令：`npm run test:security`
- 失败用例：`JSON write endpoints reject non-JSON content types`
- 期望结果：`Content-Type: text/plain` 的 JSON 写请求返回 415。
- 实际结果：服务正常解析并创建资源，返回 201。
- 影响：扩大跨站请求和协议混淆攻击面，也使接口契约不明确。
- 建议：所有读取 JSON 请求体的写接口仅接受 `application/json`，其他类型返回 415 和稳定错误码。
- 回归结果：目标用例、`npm run test:security` 和 `npm run test:api` 全部通过。

## DEF-004：HTTP 响应缺少基础安全头

- 风险级别：中
- 状态：已修复
- 修复提交：`1d29be4`
- 修复时间：2026-07-10 21:55（Asia/Shanghai）
- 测试命令：`npm run test:security`
- 失败用例：`static and API responses include baseline security headers`
- 期望结果：静态页面和 API 至少返回 `X-Content-Type-Options: nosniff`、`Content-Security-Policy` 和 `Referrer-Policy`。
- 实际结果：首个检查的 `X-Content-Type-Options` 为缺失；现有响应也未统一设置 CSP 与 Referrer-Policy。
- 影响：浏览器 MIME 嗅探、注入内容执行和引用来源泄露的防护不足。
- 建议：在统一响应层添加安全头，并为本地页面定义只允许自身脚本和样式的 CSP。
- 回归结果：目标用例、`npm run test:security`、`npm run test:api` 和 `npm run test:ui` 全部通过。
