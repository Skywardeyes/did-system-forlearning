# Agent-Executable Comprehensive Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add isolated, Agent-executable unit, integration, API, functional, security, and Chromium UI test suites to the existing DID/VC application.

**Architecture:** Keep the existing `node:test` suite and add focused layer directories plus shared fixtures. Node-based suites instantiate `createAppServer()` with temporary stores; Playwright starts a separate application process with temporary `DATA_FILE` and `LOG_FILE`, then drives Chromium and retains failure artifacts.

**Tech Stack:** Node.js 20, `node:test`, `node:assert/strict`, Fetch API, `@playwright/test`, Chromium.

## Global Constraints

- Production dependencies remain empty; only `@playwright/test` may be added to `devDependencies`.
- UI tests run Chromium only.
- Tests never read or mutate `data/store.json` or `data/logs.json`.
- Stable product/security defects remain as failing tests and are recorded in `docs/测试缺陷报告.md`; do not change product code to make them pass.
- Test-infrastructure defects must be fixed.
- Security checks target only ephemeral localhost instances and perform no destructive host/workspace actions.
- `npm run test:all` must be non-interactive and return a non-zero exit code on any failure.

- [ ] **Step 3: Create shared isolated fixtures**

Create `test/helpers/fixture.js` with imports from `node:fs/promises`, `node:os`, `node:path`, `../../src/store.js`, `../../src/log-store.js`, `../../src/log-service.js`, `../../src/vc-service.js`, and `../../src/server.js`. Implement the three produced interfaces. `startTestApp` must call `server.listen(0, '127.0.0.1')`, register `t.after()` for `server.close()`, and use the fixture's temporary log service rather than the module default.

Core implementation:

```js
export async function startTestApp(t) {
  const fixture = await createFixture(t);
  const server = createAppServer(fixture.service, { logService: fixture.logService });
  await new Promise((resolve, reject) => {
    server.once('error', reject);
    server.listen(0, '127.0.0.1', resolve);
  });
  t.after(() => new Promise((resolve) => server.close(resolve)));
  return { ...fixture, server, url: `http://127.0.0.1:${server.address().port}` };
}
```
