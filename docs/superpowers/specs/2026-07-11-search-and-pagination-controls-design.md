# 搜索与分页控件统一设计

## 目标

统一 DID、VC、验签记录和系统日志列表的搜索与分页交互，使搜索意图清晰、清除操作快捷，并让每页展示数量与当前页信息位于同一分页行。

## 适用范围

- DID 列表
- VC 列表
- 验签记录列表
- 系统日志列表

## 搜索控件

每个列表的搜索区按顺序包含：`搜索` 说明、搜索输入框、输入框内的清除 `×` 按钮，以及约 70px 宽的 `搜索` 按钮。

- 输入框的默认占位文字统一为“支持模糊搜索”。
- 清除按钮仅在输入框存在内容时显示，并具有可访问名称。
- 点击清除按钮后清空输入值、清除当前搜索条件，并从第一页刷新列表。
- 点击“搜索”按钮或在输入框内按 Enter 执行搜索，并从第一页显示结果。
- 仅输入文字不触发列表请求，避免搜索按钮与自动搜索产生重复语义。
- 系统日志保留类型、结果、级别、模块和时间范围等高级筛选条件；这些筛选项继续在变更后生效，只有文本搜索改为显式提交。

## 分页控件

每个列表的分页栏在同一行包含：`每页展示` 说明、约 80px 宽的数量下拉框、总条数和当前页信息、上一页按钮、下一页按钮。

- 下拉选项保持 10、20、50。
- 更改每页展示数量后回到第一页并刷新列表。
- 当前页信息沿用“共 N 条 · P/T 页”的表达。
- 上一页和下一页在边界页保持禁用。
- 系统日志的每页数量控件从高级筛选区移到分页栏，避免重复入口。

## 布局与响应式行为

- 桌面宽度下，搜索控件尽量保持同一行；输入框占据剩余宽度。
- 搜索按钮目标宽度约 70px，每页数量下拉框目标宽度约 80px。
- 分页栏使用弹性布局，所有分页元素在可容纳时保持同一行。
- 窄屏下允许搜索栏和分页栏按控件组换行，同时保留内边距和间距，不产生页面水平溢出。
- 所有新增控件都使用显式标签或可访问名称，清除按钮支持键盘操作。

## 组件与数据流

- HTML 提供各列表的搜索标签、输入框、清除按钮和提交按钮。
- 列表分页渲染函数接收列表类型及分页元数据，生成对应的每页数量控件和页码按钮。
- 页面控制器为搜索提交、Enter、清除、页大小和翻页操作绑定事件。
- 现有列表状态函数继续负责把搜索、页大小变化归一到第一页；不改变后端 API 和查询语义。

## 测试与验收

- 单元测试验证分页 HTML、标签、页大小选择值和按钮边界状态。
- 交互测试验证按钮/Enter 搜索、清除搜索、页大小切换回到第一页。
- Chromium UI 测试覆盖四类列表的文案、控件宽度、同排布局和清除按钮行为。
- 移动端测试验证控件可换行、可操作且无水平溢出。
- 执行完整测试套件，并在 `test-records` 中生成包含测试时间和简要分析的新记录。

## 非目标

- 不改变模糊搜索的后端匹配算法。
- 不增加新的每页数量选项。
- 不改造系统日志除文本搜索和页大小位置之外的高级筛选行为。
