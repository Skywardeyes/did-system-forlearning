# 本地测试证据链设计

## 目标

增强现有本地测试记录器，使 AI 触发的每次测试都产生永久保留、可追踪、可复核、可检测篡改的证据包。AI 只解释结果，退出码、测试框架输出和证据文件决定成功或失败。

## 执行与判定

`npm run test:evidence` 依次执行 Node 和 Chromium UI；Node 失败时仍继续执行 UI。只有两个阶段退出码均为零、结果可解析、失败/跳过/todo 均为零且证据落盘成功，批次才标记通过。测试失败或证据生成失败均返回非零退出码。

工作区不干净时允许执行，但元数据必须标记包含未提交修改，不能把结果解释为纯 Git 提交的验证结果。

## 永久证据包

每次运行创建不覆盖的 `test-records/YYYY-MM-DD/<时间+08-00>/`，不提供自动删除或保留上限。批次至少包含：

- `metadata.json`：批次、Git、工作区、环境、命令、时间、耗时和退出码；
- `result.md`：中文摘要；
- `node.log`、`ui.log`：原始完整输出；
- `playwright-report/`：本次 HTML 报告；
- `test-results/`：截图、视频、Trace 和错误上下文；
- `test-cases.json`：实际发现的用例、测试类型、文件位置和结果；
- `evidence-manifest.json`：证据文件相对路径、字节数和 SHA-256；
- `checksums.sha256`：通用哈希清单。

成功批次也归档 Playwright HTML 报告；无失败附件时保留空的 `test-results/` 目录，并在清单中记录目录状态。

## 用例映射

测试框架输出解析为用例级记录。Node TAP 和 Playwright 测试分别记录标题、结果和可解析的文件位置；映射规则不得凭模块相似性推断。无法定位文件时明确写 `null`，不得伪造路径。

## 本地命令

- `npm run test:evidence`：执行并生成批次证据；
- `npm run test:evidence:verify -- <批次目录>`：重新计算 SHA-256，报告缺失、修改和新增文件；
- `npm run test:evidence:list`：按时间倒序列出所有完整批次的提交、工作区状态和结果。

原有 `npm run test:all` 保持兼容并指向同一证据执行器。

## 哈希与验证

生成清单前关闭所有报告写入。哈希清单不包含自身，`evidence-manifest.json` 可包含其他证据文件但不包含自身哈希，避免循环依赖。验证命令必须对缺失、内容变化和清单外新增文件分别报告，并以非零退出码表示不一致。

## 安全与隐私

证据包不保存环境变量、令牌、私钥或完整请求凭据。原始测试输出若出现敏感字段，执行器在落盘前应用现有敏感字段规则；报告不得复制业务数据文件。`test-records/` 保持在 `.gitignore` 中，仅保存在本机。

## 验收标准

- 成功与失败测试都生成完整证据包。
- 失败批次仍执行两个阶段并返回非零。
- 成功批次包含 HTML 报告，用例清单与测试数量一致。
- 修改任一受保护文件后验证命令失败并指出文件。
- 历史列表包含全部批次且无自动删除。
- 自动化测试覆盖生成、失败路径、哈希验证、唯一目录和永久留存行为。
