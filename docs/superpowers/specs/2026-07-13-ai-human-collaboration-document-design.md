# AI 与人工协作过程说明文档设计

## 目标

新建 `docs/AI与人工协作过程说明.md`，将用户提供的十个步骤扩展为适合答辩、项目复盘和交付说明的正式文档。

## 文档结构

1. 文档目的与核心结论。
2. 参与角色和职责：人工需求方/评审者、需求分析 AI、开发 Agent、测试 Agent。
3. 十个阶段的详细协作过程。
4. 开发 Agent 与测试 Agent 的任务隔离、交付接口和冲突处理。
5. 需求、技术选型、代码、测试用例、缺陷和验收证据的追踪链。
6. 人工不可替代的决策与 AI 不应独立承担的工作。
7. 完整流程图与阶段回退机制。
8. 项目真实案例：生命周期、日志、DID Method、测试补充、缺陷修复和人工验收。
9. 方法总结和答辩表述。

## 十个阶段的统一模板

每个阶段包含：

- 阶段目标；
- 输入材料；
- AI/Agent 工作；
- 人工工作；
- 主要输出；
- 人工确认点；
- 不通过时的回退方式。

## 事实与表述原则

- 使用“AI”表示需求分析、方案整理和协作编排；使用“Agent”表示有明确任务边界的实施单元。
- 不将 AI 描述为需求范围、业务优先级和最终验收的独立责任人。
- 开发 Agent 与测试 Agent 的“分别生成”不表示完全无协作；两者共享已确认需求和接口契约，但测试 Agent 不应仅复制开发 Agent 的实现假设。
- 使用当前 DID/VC 项目的真实细节作为例证，不虚构人员、时间和具体对话。
- 强调每个人工确认点都是质量门禁，未通过时回退到相应的需求、设计、测试或实现阶段。

## 验收标准

- 文档包含完整的十阶段详细说明，而不是只对原句扩写。
- 每个阶段都能识别输入、职责、输出和确认点。
- 包含至少一个可直接在 Markdown 中渲染的 Mermaid 流程图。
- 包含需求到交付的追踪表。
- 包含 DID/VC 项目中的真实协作与纠偏案例。
- 明确区分人工决策责任与 AI/Agent 执行责任。
- 文档无空章节、占位符、重复标题或层级跳跃。
