# AI 与人工协作过程 HTML PPT 设计说明

## 1. 目标

将 `docs/AI与人工协作过程说明.md` 转换为适合项目答辩和过程汇报的单文件 HTML 演示文稿。演示需要在有限页数内说明人工与 AI/Agent 的职责边界、十阶段协作闭环、项目工程案例、可追踪证据和最终结论。

## 2. 受众与使用场景

- 主要受众：毕业设计或课程项目答辩老师、项目评审者。
- 主要场景：浏览器全屏放映，也可在桌面或移动端滚动检查细节。
- 演讲目标：使听众理解项目不是由 AI 一次性生成，而是经过人工决策、Agent 分工、自动化验证和人工验收的可追踪工程过程。

## 3. 内容策略

采用“平衡型答辩方案”，预计 13 页左右，不逐段照搬源文档。十个阶段按照工程目的聚合为五段叙事：

1. 需求形成：阶段 1—3；
2. 方案确认：阶段 4—5；
3. 并行实现：阶段 6；
4. 测试与修复闭环：阶段 7—9；
5. 人工验收：阶段 10。

关键内容包括：

- 核心观点：AI 提高分析、实施和验证效率，人工控制目标、边界、风险和最终质量；
- 人工、需求分析 AI、开发 Agent、测试 Agent 的职责与禁止越界事项；
- 十阶段完整流程、阶段门禁与失败回退；
- 开发 Agent 与测试 Agent 的共享契约和视角隔离；
- DID Method 纠偏、日志需求完善、测试覆盖补充、并发写入缺陷、真实浏览器验收五个项目案例；
- 需求—设计—实现—测试—人工验收的追踪关系；
- 人工不可替代的判断与 AI/Agent 的工作边界；
- 面向答辩的最终总结。

## 4. 页面结构

建议页面顺序如下：

1. 封面；
2. 一句话核心结论与协作原则；
3. 四类角色与职责边界；
4. 十阶段流程总览；
5. 阶段 1—3：需求从初稿到权威基线；
6. 阶段 4—5：技术与测试方案双线确认；
7. 阶段 6：开发 Agent 与测试 Agent 并行协作；
8. 阶段 7—9：覆盖评审、执行验证、缺陷修复闭环；
9. 阶段 10：自动化达标后的人工验收；
10. 项目案例一：DID Method 与日志需求纠偏；
11. 项目案例二：测试覆盖、并发缺陷与浏览器验收；
12. 需求到交付的追踪链路与证据；
13. 人工不可替代的决策、AI 边界与答辩总结。

## 5. 视觉与交互设计

- 采用深色科技感答辩主题，使用蓝色与青绿色表达 AI/自动化，暖色表达人工决策和确认门禁。
- 保持单一画布式构图，避免密集仪表盘风格；每页突出一个主要结论。
- 使用 CSS 绘制流程、阶段编号和追踪链路，不依赖外部图片或网络资源。
- 提供上一页、下一页、目录、全屏和进度显示。
- 支持键盘方向键、PageUp/PageDown、Home/End 和数字目录跳转。
- 工程细节放入原生 `details/summary` 下拉区，展开时不破坏页面导航。
- 支持 `prefers-reduced-motion`，保证焦点样式、语义标签和足够颜色对比度。
- 桌面端按 16:9 幻灯片展示；窄屏下允许页内纵向滚动，避免内容裁切。

## 6. 技术架构

交付物为一个独立 HTML 文件：

- HTML：每页使用语义化 `section`，目录使用导航列表；
- CSS：主题变量、响应式布局、幻灯片切换、打印样式和无动画偏好；
- JavaScript：维护当前页索引，统一处理按钮、键盘、目录、URL hash 和全屏；
- 内容来源：仅使用 `docs/AI与人工协作过程说明.md` 和项目内已验证的工程事实，不新增未经证实的数据。

## 7. 异常与边界处理

- 页码越界时固定在第一页或最后一页；
- URL hash 无效时回到封面；
- 输入控件、链接或展开区域获得焦点时，键盘操作不误触翻页；
- 浏览器不支持全屏 API 时隐藏或禁用全屏能力，不影响基本放映；
- 禁用 JavaScript 时仍能按文档顺序阅读全部内容；
- 打印时展开为连续页面，隐藏导航控件。

## 8. 验收标准

- 生成 `outputs/ai-human-collaboration-presentation.html`，无需服务器即可打开；
- 页面数量与目录目标一致，所有导航目标有效；
- 十个协作阶段均被覆盖且顺序正确；
- 五个真实项目案例、职责边界、追踪关系和答辩总结均出现；
- HTML 内不含外部依赖、占位文本或生成过程说明；
- JavaScript 可解析，键盘、按钮、目录、URL hash 和全屏逻辑可用；
- 桌面与窄屏布局无明显溢出、遮挡或无法阅读内容；
- 自动化结构测试和项目 Node 回归测试通过；
- 最终使用真实浏览器逐页检查，并验证至少一个窄屏视口。

## 9. 交付范围

本次只新增 HTML PPT、相应自动化测试、设计说明和实施计划。不修改源 Markdown，也不修改现有两份 HTML PPT 或项目业务代码。
