# DID/VC 测试答辩 HTML 演示稿设计

## 目标

将《DID-VC 系统测试用例表-高覆盖版》转化为适合课程答辩的交互式 HTML 演示稿。演示稿同时支持一页一屏的 PPT 式纵向切换，以及页内测试证据的折叠展开。

## 受众与使用方式

- 主要受众：课程答辩老师、项目验收人员。
- 主要场景：笔记本或投影仪全屏演示。
- 次要场景：老师在演示后自由浏览并展开具体用例。
- 交付形式：可以直接在浏览器打开的单文件 HTML。

## 内容结构

演示稿预计包含 12 个全屏章节：

1. 封面：DID/VC 系统高覆盖测试与可信验证。
2. 总览：208 条用例、六类测试及执行结果。
3. 测试体系：单元、集成、接口、功能、兼容和安全的分层关系。
4. DID 身份与 Method：Issuer、Holder、`did:example` 和 `did:key`。
5. DID 生命周期：更新、版本冲突、密钥轮换和不可逆停用。
6. VC 签发：正常签发、角色校验、参数边界和私钥保护。
7. 七项完整验证：格式、Issuer、DID 状态、密钥版本、签名、有效期和凭证状态。
8. VC 生命周期：暂停、恢复、替换、撤销和过期。
9. 台账、搜索与日志：分页、失败原因、审计关联和日志清理。
10. 安全边界：泄密、XSS、原型污染、目录穿越和 HTTP 加固。
11. UI 与可信证据：桌面/移动兼容、空状态、失败产物和校验和。
12. 结论与边界：已覆盖能力、生产级未覆盖项和答辩总结。

## 交互设计

- 页面使用纵向滚动吸附，每个章节默认占满一屏。
- 支持鼠标滚轮、上下方向键、Page Up/Page Down 和侧边章节导航。
- 当前章节在侧边导航中高亮，页面显示章节序号与总页数。
- 每个主题页包含 2–4 个折叠卡片，默认仅显示结论，展开后显示典型用例、步骤、预期结果与答辩解释。
- 折叠卡片使用原生 `details/summary`，以保持键盘可访问性。
- 提供全屏按钮、返回封面按钮和演讲进度条。
- 在用户展开较长内容时，允许当前页内部滚动，避免内容裁切。

## 视觉设计

- 整体为深蓝技术答辩风格，主背景使用深蓝灰，辅以青色和绿色。
- 青色表示 DID/密钥/技术链路，绿色表示测试通过，黄色表示边界与提醒，红色仅用于失败或攻击场景。
- 封面与总览使用大号数据、简化环形和分层卡片，不使用无关装饰图片。
- 正文保持投影可读：大标题、短结论、有限的并列卡片，详细文字放入折叠区。
- 移动端将侧边导航折叠为顶部进度控件，卡片改为单列。

## 数据口径

- 总用例数：208。
- 分类：接口 13、单元 62、集成 21、功能 87、兼容 17、安全 8。
- 工作簿中各类用例均记录为通过。
- 同时解释“208 条验收场景”与“自动化 `test()` 数量”属于不同统计口径。
- 具体案例来自现有高覆盖工作簿、测试代码与项目文档，不伪造生产级认证结论。

## 内容边界

演示稿会明确标注尚未覆盖：

- 正式 W3C cryptosuite 跨系统互操作。
- 公网部署、TLS 和完整权限体系。
- 生产级 KMS/HSM 密钥托管。
- 压力、拒绝服务与完整多浏览器矩阵。
- 真实 DID Resolver、链上注册或标准状态列表。

## 验收标准

- 单文件 HTML 可直接打开，不依赖网络资源。
- 12 个章节可通过滚轮、键盘和导航切换。
- 折叠卡片可使用鼠标和键盘操作。
- 桌面和移动宽度下不产生不可控的整页横向滚动。
- 所有核心统计与工作簿一致。
- 每个主要测试域都包含至少一个典型正常用例和一个边界/失败用例。
- 页面明确区分“已测试范围”与“未达到的生产级能力”。
