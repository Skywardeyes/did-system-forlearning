# 盒模型统一留白设计

## 目标

检查并调整 Web 界面全部主要盒模型，确保可读内容和可交互元素不会直接贴住容器边缘，同时保留现有的通栏标题、表格和代码区视觉结构。

## 现状

静态 CSS 与真实浏览器计算样式显示：

- `.list-tools` 左右内边距为 0，搜索框和页大小控件贴近面板边缘。
- `.pagination` 四周内边距为 0，分页控件贴近面板右侧和底部。
- `.panel` 自身无内边距，依赖子区域分别管理留白。
- 移动端 `.nav-item` 仅有 2px 内边距，视觉留白不足。
- 统计卡片、流程节点、表单、表格单元格、身份卡片、日志筛选、验证项、代码内容和弹窗标题已有明确内边距。

## 设计原则

- 桌面端主要内容区水平留白统一为 20px。
- 移动端主要内容区水平留白为 12–14px。
- 小型标签、徽标、按钮等控件内部留白不小于 6px。
- 通栏标题、表格、代码块可以连接面板边框，但其中的文字和控件必须通过自身内边距与边缘分离。
- 不为 `.panel` 直接添加统一 padding，避免标题分隔线、表格和深色代码区出现双重缩进。
- 不改变现有信息架构、颜色、字体、数据逻辑和交互流程。

## 调整范围

### 列表工具栏

`.list-tools` 使用 `padding: 12px 20px`，移除依赖外边距制造间隔的方式。工具栏内部继续使用 flex gap；输入框和选择框保留自身内边距。

### 分页栏

`.pagination` 使用 `padding: 0 20px 16px`。分页文字和按钮与面板右侧、底部保持明确距离；分页按钮补充最小点击区域与内部留白。

### 操作按钮

`.table-action` 和面板内 `.text-button` 使用至少 6px 的垂直、水平点击留白，避免文字直接贴着自身点击盒边缘。侧边栏重置按钮单独保留紧凑布局，但仍提供最小垂直留白。

### 移动导航

移动端导航按钮从 2px 调整为至少 `6px 4px`，并在 62px 高度内保持五个入口可见、等宽、无溢出。

### 通栏区域

- `.section-heading` 保持 `16px 20px`。
- `th, td` 保持 `13px 18px`。
- `.code-panel pre` 保持 20px。
- `.log-filters` 保持 `12px 20px`，移动端保持 12px。
- `.activity-item`、`.identity-card`、`.check-item` 保持现有明确内边距。

## 自动化检查

扩展 `test/ui-layout.test.js`，验证：

- `.list-tools` 有桌面 `12px 20px` 内边距。
- `.pagination` 有水平和底部内边距。
- `.table-action` 与 `.text-button` 具有非零点击内边距。
- 移动端 `.nav-item` 内边距不少于 4px。
- 面板通栏子区域各自具有内容内边距，不要求 `.panel` 本身加 padding。

扩展 Chromium UI 测试，在桌面和 390×844 窄屏下读取关键元素计算样式，确认关键内容容器留白符合规则、页面无水平溢出、五个移动导航入口仍可见和可操作。

## 验收标准

- 搜索、分页和操作按钮不再贴住面板或自身盒模型边缘。
- 桌面端主要内容区使用一致的 20px 水平留白。
- 移动端主要内容区使用 12–14px 水平留白，导航按钮至少 4px 内边距。
- 标题、表格和代码区保持通栏，但内容均有明确内边距。
- Chromium 桌面与窄屏检查通过。
- 全量自动化测试通过，并生成新的测试记录。
