## 文档分工

科班内容细纲更新技术事实、测试分层、运行命令、UI 入口和局限；科班流程大纲只更新演示动作、页面位置和观察结果；非科班双人演讲细纲以易懂语言更新操作步骤、串词、备选话术、测试数字和检查清单。

保留现有时间轴、章节顺序、A/B 分工和整体演示节奏。

## 删除或改写的过期内容

- 删除 Playwright 未安装、UI 未执行、正式汇报前安装 Playwright等表述。
- 删除 UI 测试依赖仍需处理的局限。
- 将旧的 120/120 测试结果更新为当前结果。
- 将只引导返回总览台账的步骤，按场景改为签发页台账或验证页记录台账。
- 需要保留 `npm test` 入口风险时，明确推荐正式命令为 `npm run test:node` 和 `npm run test:ui`，不得将其描述为 Playwright 缺失。

## 保留的项目限制

教学实现、非生产级 W3C 互操作方案、本地 JSON 未加密、无账户认证和权限控制、无真实身份核验、未接入跨机构网络。

## 验收标准

- 三份文档不再出现 120 项、Playwright 未安装或 UI 未执行。
- 三份文档中的测试数字和演示入口一致。
- 新增台账和失败原因均在对应演示步骤中出现。
- 不改变时间轴和 A/B 分工。
- Markdown 与 Mermaid 结构保持完整，`git diff --check` 通过。
