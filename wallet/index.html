SELF-CUSTODY HOLDER WALLET

信证钱包

Holder 私钥仅在当前浏览器的本地 IndexedDB 中保存；钱包不会调用平台接口上传私钥，也不会把私钥写入凭证交付包。

1. 创建我的 DID

在本设备本地生成不可导出的 Ed25519 私钥和 `did:key`。请妥善保管本浏览器配置；此 MVP 尚未提供跨设备恢复。

2. 向签发机构登记

把“登记包”交给学校或企业。机构只登记 DID Document 中的公钥，不能取得你的私钥。

登记包仅含 DID、公钥和 DID Document；不含私钥、恢复材料或已有凭证。

3. 接收机构签发的 VC

从签发平台获得“钱包 VC 交付包”后粘贴到这里。该包包含完整 VC 和由 Issuer 签名的 SD-JWT 披露材料，只会保存到本钱包本地。

4. 本地生成最小披露证明

验证方应提供一次性的 Challenge 和自身域名/标识。钱包在本地选择字段、组装 SD-JWT，并用 Holder 私钥对本次出示绑定签名。

提示：本 MVP 已实现 Holder 本地签名绑定；验证方的 Challenge 应由验证方保存并只接受一次，服务端防重放台账将在下一阶段补充。
