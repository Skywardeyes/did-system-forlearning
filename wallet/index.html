在此设备的浏览器中创建并管理 Holder DID。Ed25519 私钥以不可导出 CryptoKey 保存于 IndexedDB，不会被发送至签发平台。

UNIFIED ACCOUNT

登录个人账号，登记 Holder DID

这里上传的只有公开登记包。私钥继续保存在本机 IndexedDB，不会进入平台数据库。

登录令牌只保存在当前页面内存；登记后会清空密码输入框。组织需要先关联这个公开 DID，才能向它签发 VC。

CREATE IDENTITY

创建本地 DID

一个浏览器配置只保存一个主身份；重新创建会替换当前钱包身份。

MANUAL FALLBACK

公开登记包

如机构暂未接入钱包登记协议，可复制这份公开登记包交给机构；其中不含私钥、恢复材料或任何已持有凭证。

凭证收件箱

签发机构创建凭证后，钱包会自动发现待领取消息。领取操作由本地 Holder 私钥签名，成功后凭证直接导入本地库。

钱包打开后每 15 秒自动检查一次。关闭网页后的后台推送属于下一阶段 Web Push / 移动端能力。

本地凭证库

已领取的 VC 与 SD-JWT 披露材料仅保存在当前浏览器的钱包 IndexedDB。可作为兼容旧流程的备用方式手动导入交付包。

MANUAL IMPORT

导入钱包交付包

LOCAL VAULT

已持有凭证

这些凭证不会被回传给平台。可从多张凭证分别勾选字段，再生成一次组合出示证明。

选择性披露

钱包在本地仅组合你勾选的声明，并用 Holder 私钥对本次出示的 Challenge 与验证方域名进行签名绑定。

MULTI-CREDENTIAL DISCLOSURE

从多张凭证中选择字段

每个字段始终保留所属凭证和签发机构，最终由 Holder 私钥对整个组合一次签名。

请使用验证方提供的一次性 Challenge。组合中任意凭证或字段被增删、替换，Holder 整体签名都会失效。
